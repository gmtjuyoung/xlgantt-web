## Design Context

### Users
- 프로젝트 PM, 팀원, 관리자
- 주로 데스크톱에서 주간 계획/진척/배정을 빠르게 확인하고 업데이트
- 제한된 시간 내에 상태 판단과 의사결정을 해야 하는 운영/실행 환경

### Brand Personality
- 실무형, 명확함, 신뢰감
- 감성보다 가독성과 즉시 판단을 우선
- 복잡한 정보를 빠르게 정리해 보여주는 톤

### Aesthetic Direction
- 데이터 밀도 높은 B2B 운영 대시보드
- 라이트 테마 중심, 강한 대비와 구획 분리
- 장식 최소화, 상태 색과 구조적 리듬으로 정보 계층 전달

### Design Principles
- 상태는 색과 형태를 동시에 사용해 즉시 식별 가능해야 한다.
- 행/블록 단위는 가로 분리선과 배경 톤으로 명확히 분리한다.
- 중요한 값(진척률/기간/WBS)은 단색 대비와 고정폭 숫자로 읽기 쉽게 유지한다.
- 인터랙션 요소(토글/정렬/뷰 전환)는 기능 의도가 즉시 보이게 표현한다.
- 같은 의미의 UI는 화면마다 동일한 시각 규칙을 따른다.
